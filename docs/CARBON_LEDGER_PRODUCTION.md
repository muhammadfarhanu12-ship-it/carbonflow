# Carbon Ledger Production Usage

The Carbon Ledger is the audit-facing module for Scope 1, Scope 2, and Scope 3 activity records, approval workflow, supplier-linked emissions, financial exposure, CSV import, and report generation.

## Required Configuration

- Frontend API calls must use `VITE_API_URL` with the `/api` prefix.
- Backend APIs require authentication and company scoping.
- Sample factors are fallback placeholders only and must not be presented as official factors.

## Emission Factors

Create official or custom emission factors before formal reporting. Factor matching prefers active company-specific factors, then active global factors, and only falls back to clearly marked sample factors when no database factor exists.

Matching considers scope, category, activity type, activity unit, factor key, country, region, active status, effective dates, source year, version, and company scope.

## Recording Activity

Use the Carbon Ledger activity form to record Scope 1, 2, or 3 activity. A calculated record requires positive activity amount, activity unit, reporting period, activity date, and a matched or provided factor. Draft records may be saved for follow-up, but are excluded from approved emissions totals.

Stored calculation basis:
- `emissions = activityAmount x emissionFactor`
- factor value/unit/source/year/version used at calculation time
- kgCO2e and tCO2e
- sample, official, or custom factor status
- calculation status

## Approval Workflow

Statuses are `draft`, `submitted`, `reviewed`, `approved`, `rejected`, `needs_correction`, and `archived`.

Approved records are used by default for enterprise reporting. Draft and archived records are excluded from approved totals. Status changes write audit logs with actor and request metadata.

## CSV Import

The CSV template includes:

```csv
scope,category,activityType,activityAmount,activityUnit,factorKey,reportingPeriodStart,reportingPeriodEnd,activityDate,facility,businessUnit,country,region,supplier,notes
```

The backend validates every row, matches emission factors, returns row-level errors, flags missing/sample factors, and saves only valid rows after confirmation.

## Supplier Linking

Use the supplier picker to link records by `supplierId`. The backend validates that the supplier belongs to the same company and stores supplier snapshot metadata for audit history. Records with only supplier text metadata appear as unverified supplier links.

## Financial Ledger

Financial entries can link to emission records, shipments, and suppliers. They track logistics cost, carbon tax, offset cost, internal carbon price, total cost, currency, and creator/updater metadata.

## Reports

Generate PDF or CSV reports from the Carbon Ledger. The default is approved records only. All-record reports are allowed with warnings for sample factors, missing factors, unapproved records, zero amounts, and calculation errors. Downloads use authenticated blob requests.

Reports include company name, reporting period, generated by/at metadata, inclusion policy, emissions totals, Scope 1/2/3 breakdown, categories, suppliers, monthly breakdown, data quality notes, methodology, calculation basis, and status summary.

## Production Smoke Checklist

Automated non-mutating smoke check:

```powershell
cd backend
node scripts/smoke-carbon-ledger.js
```

Controlled mutating smoke check for staging or a dedicated production test company only:

```powershell
$env:SMOKE_TEST_EMAIL="test-admin@example.com"
$env:SMOKE_TEST_PASSWORD="<password>"
$env:SMOKE_RUN_MUTATING_TESTS="true"
node scripts/smoke-carbon-ledger.js
```

Optional URL overrides:

```powershell
$env:SMOKE_FRONTEND_URL="https://carbonflow-nu.vercel.app"
$env:SMOKE_BACKEND_URL="https://carbonflow-h9cj.onrender.com"
$env:SMOKE_API_BASE="https://carbonflow-h9cj.onrender.com/api"
```

## Known Limitations

- Sample factors remain available only as clearly marked fallback placeholders.
- Formal enterprise assurance still requires a verified factor library, production smoke execution, retention policy sign-off, and security/compliance review.
